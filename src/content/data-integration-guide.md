> 欢迎使用。这个向导面向第一次接触本产品的用户，目标是让你在 10~15 分钟内看到可用的数据流。  
> 建议按左侧顺序完成，每一步完成后再进入下一步。

### 1) 第一次打开后你会看到什么

- 顶部：全局搜索、刷新、设置入口
- 左侧：群消息流（Telegram / WeChat）
- 右侧：信息流（X / 新闻等）
- 设置页：所有数据源都在这里接入和管理

如果页面能正常打开，但没有实时数据，这是正常的；你还没有接入数据源。

### 2) 最快可用路径（推荐顺序）

建议按这个顺序接入，成功率最高：

1. WeChat（本地）  
2. Telegram（账号登录）  
3. X / Twitter（WSS）  
4. BlockBeats（新闻）

每接好一个数据源，都可以立即回到主界面观察是否有数据出现。

### 3) WeChat（本地敏感数据，必须本机）

WeChat 是敏感链路：请只在本机运行后端，不要部署到公网。

本机先启动服务（示例）：

```bash
python main.py
```

然后在设置页填写：

- 启用微信监控：开启
- 服务地址：`http://localhost:5678`
- 轮询间隔：建议 2000~5000ms

点“测试连接”通过后，到主界面左侧切换 `WX`，添加群卡片即可看到实时消息。

### 4) Telegram（个人账号接入）

你需要准备：

- API ID
- API Hash
- 登录手机号

操作顺序：

1. 在设置页填写 API ID / API Hash，点击“连接”
2. 填手机号，点击“登录”
3. 按提示输入验证码（如开启 2FA 还需密码）

状态显示 `connected` 即表示接入成功。

### 5) X / Twitter（WSS 推送）

你需要有可用的推送服务地址和鉴权 Token。

设置页填写：

- 启用 WSS 推送：开启
- WSS 地址：`wss://...`
- Token：你的访问令牌

成功后，右侧 `X` 标签会持续出现新卡片。

### 6) BlockBeats 新闻

你只需要一个可用 API Key。

设置页填写：

- 启用新闻监控：开启
- API Key：填入你的 key

成功后，右侧新闻标签会自动刷新内容。

### 7) 常见问题（新用户版）

- 页面正常但没数据：通常是还没完成数据源接入
- WeChat 连接失败：先浏览器访问 `http://localhost:5678/api/history?limit=1` 自检
- Telegram 报错：优先检查 API ID / API Hash 和 2FA
- Twitter 无推送：检查 WSS 地址、Token、服务端是否返回 connected

### 8) 运行形态说明

- 桌面版：支持本地桥接能力（WeChat / Telegram / WSS）
- Web 版：页面可运行，但本地桥接能力需要对应远端 API（WeChat 仍建议只本地）

> 建议：先用桌面版跑通全链路，再考虑 Web 部署策略。
